# 🤝 Contributing to PyBe

Thank you for contributing to **PyBe MERN App**!

This project follows a collaborative GitHub workflow. Please read these guidelines before making any changes.

---

# 📌 Contribution Workflow

1. Make sure you have access to the repository.
2. Sync your local repository with the latest `develop` branch.
3. Create a new branch for your task.
4. Make your changes.
5. Commit your changes using meaningful commit messages.
6. Push your branch to GitHub.
7. Open a Pull Request (PR) to the `develop` branch.
8. Wait for review and approval before merging.

---

# 🌿 Branch Naming Convention

Create branches using the following format:

### New Feature

```text
feature/login-page
feature/dashboard
feature/session
```

### Bug Fix

```text
fix/navbar
fix/api
```

### Documentation

```text
docs/readme
docs/api
```

---

# ✅ Commit Message Convention

Use meaningful commit messages.

Examples:

```text
feat: add dashboard page
feat: implement session module
fix: resolve navbar issue
docs: update README
style: improve landing page UI
refactor: optimize API logic
```

Avoid commit messages like:

```text
update
final
changes
done
test
```

---

# 🚫 Repository Rules

- Do NOT commit directly to the `main` branch.
- Always create a new branch from `develop`.
- Open a Pull Request for every change.
- Wait for review before merging.
- Keep commits small and meaningful.
- Update documentation whenever necessary.

---

# 🔀 Pull Request Checklist

Before creating a Pull Request, make sure:

- [ ] Code has been tested.
- [ ] No unnecessary files are included.
- [ ] No merge conflicts exist.
- [ ] Documentation is updated (if required).
- [ ] Commit messages follow the project convention.

---

# 💬 Need Help?

If you are unsure about anything, contact the project maintainer before making major changes.

Happy Coding! 🚀
